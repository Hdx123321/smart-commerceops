# Smart CommerceOps 术语速查

**日期**: 2026-06-12
**分类**: 概念 / 全栈

## 前后端基础

- **JWT (JSON Web Token)** — 前后端间安全传递身份的令牌。三段 base64 编码：Header（签名算法）+ Payload（用户 ID/角色/过期时间）+ Signature（服务端密钥签名）。任何人都能读前两段，但只有服务端能验证签名。项目存在 `localStorage` 键 `smart-commerceops-token`。

- **localStorage** — 浏览器提供的键值存储，数据不随标签页关闭而丢失。适合存 token 等需持久化的数据。与 sessionStorage 的区别：localStorage 永久存储，sessionStorage 关闭标签页即清。

- **CORS (跨域资源共享)** — 浏览器安全机制。前端 `localhost:5173` 和后端 `localhost:8090` 属于不同"源"（协议+域名+端口任一不同就算跨域）。Gateway 统一配置 CORS 允许前端域名访问。

- **Hook (React)** — 让函数组件拥有状态和副作用能力。`useState` 存状态、`useEffect` 发请求/操作 DOM、自定义 Hook 复用逻辑。

- **useQuery** — TanStack Query 的读数据 Hook。只需提供 `queryKey`（缓存键名）和 `queryFn`（获取函数），自动管理 loading/error/success 三态，带缓存和自动重取。

- **useMutation** — TanStack Query 的写数据 Hook。手调用 `.mutate()` 触发，用于 POST/PUT/DELETE。不会自动执行。

## 后端与架构

- **Gateway (API 网关)** — 所有前端请求的统一入口。根据路径前缀（`/auth/**`、`/products/**`、`/cart/**` 等）转发到不同微服务。好处：前端只需记一个地址、加服务不改前端、CORS 一处配置。

- **微服务 (Microservice)** — 把大应用拆成多个独立小服务，各有自己的数据库和端口，通过网络互相通信。本项目有 5 个微服务：gateway + identity + catalog + order + analytics。

- **微服务间通信** — 本项目用 Spring `RestClient` 直接发 HTTP 请求（如 order-service 调 catalog-service 查商品信息）。不走 Gateway。其他方式还有消息队列（RabbitMQ/Kafka）。

- **@Transactional (事务)** — Spring 注解。保证方法中所有数据库操作要么全成功提交，要么失败全回滚。Checkout 中用事务保证"创建订单+扣库存+清空购物车"的原子性。

- **BCrypt** — 密码哈希算法。特点：相同密码产生不同哈希、计算速度故意慢（防暴力破解）、自带盐值。数据库永远不存明文密码。

- **DTO (Data Transfer Object)** — 层间传数据的简单对象。如 `AuthResponse`、`CheckoutRequest`——不带业务逻辑，只是数据的"快递箱"。

## 电商业务术语

- **GMV (Gross Merchandise Volume)** — 商品交易总额。电商最核心的营收指标之一。

- **AOV (Average Order Value)** — 平均每单金额 = GMV ÷ 订单数。

- **SKU (Stock Keeping Unit)** — 库存量单位。一个商品的不同规格（颜色/尺寸）对应不同 SKU。

- **低库存预警 (Low Stock Alert)** — 当商品库存量低于设定的阈值（`lowStockThreshold`）时触发提醒，需要补货。
