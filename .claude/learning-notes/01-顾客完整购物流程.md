# 顾客完整购物流程 — 从登录到 Checkout

**日期**: 2026-06-12
**分类**: 全栈 / 系统架构
**项目**: Smart CommerceOps

## 核心问题

一个顾客从打开网站到完成下单，整个应用（前端 + Gateway + 后端微服务）都发生了什么？

## 流程概览

```
浏览器 → Gateway (8090) → 微服务集群
                                ├─ identity-service (8092) — 登录/注册
                                ├─ catalog-service (8093)  — 商品
                                ├─ order-service (8094)    — 购物车/订单
                                └─ analytics-service (8095)— 数据看板
```

## 第 1 步：打开网页 — App.tsx 的门卫逻辑

访问 `http://localhost:3000`，[App.tsx](frontend/src/App.tsx) 加载：

1. `currentUser()` 从 localStorage 读 `smart-commerceops-user`
2. 首次访问 → `null` → 导航栏显示 Products + Login 按钮
3. 已登录 → 显示用户名/角色/Logout + Cart/Orders 链接

关键：用 localStorage 存 token，避免每次刷新都重新登录。这个 token 相当于浏览器的"临时门禁卡"。

## 第 2 步：注册/登录 — JWT 令牌的诞生

[LoginPage.tsx](frontend/src/pages/LoginPage.tsx)

```
POST /auth/register { username, email, password, role }
  → Gateway 路由 /auth/** → identity-service (8092)
  → AuthController.register()
    1. 检查用户名/邮箱是否已存在 → 冲突返回 409
    2. BCrypt 加密密码 → 数据库永不存明文
    3. 存入 user_accounts 表
    4. 生成 JWT Token → 返回 { accessToken, user }
  ← 前端 saveSession() → localStorage 存 token + user
  ← onAuthenticated() → 全局状态更新 → 界面刷新
```

登录流程类似，但多了密码验证步骤：`passwordEncoder.matches(明文, 数据库中的哈希)`。

## 第 3 步：浏览商品

[ProductsPage.tsx](frontend/src/pages/ProductsPage.tsx)

```
GET /products
  → Gateway → catalog-service (8093)
  → ProductController → 查 products 表 → Product[]
  ← 前端 useQuery({ queryKey: ['products'], queryFn: catalogApi.products })
  ← Ant Design Card 网格渲染
```

useQuery（TanStack Query）自动管理加载中/成功/失败三种状态，带缓存。

## 第 4 步：加入购物车

点击 Add 按钮：

```
POST /cart/items { userId, productId, quantity: 1 }
  → Gateway → order-service (8094)
  → OrderController.addToCart()
    1. 调用 catalog-service GET /products/{id} 获取商品信息
    2. 查 cart_items：该用户是否已有此商品？
       → 有：数量 +1
       → 无：新建记录
    3. 保存 → 返回 CartItem
  ← useMutation → onSuccess: Toast "Added to cart"
```

注意：order-service 主动调用 catalog-service —— 微服务间通信，不经 Gateway。

## 第 5 步：查看购物车

[CartPage.tsx](frontend/src/pages/CartPage.tsx)

ProtectedRoute 检查：已登录？是 CUSTOMER？→ 通过才渲染。

```
GET /cart/{userId} → order-service → CartItem[]
  ← 前端渲染表格：商品/数量/单价/小计 + 总价
  ← 总价在前端计算：reduce((sum, item) => sum + item.unitPrice * item.quantity, 0)
```

总价展示用前端计算即可，但真正下单时后端会重新计算（防篡改）。

## 第 6 步：Checkout — 最关键的步骤

填写 shippingAddress + phoneNumber，点 Checkout：

```
POST /checkout { userId, shippingAddress, phoneNumber }
  → Gateway → order-service (8094)
  → OrderController.checkout()  ← @Transactional 事务!!
    1. 检查购物车 → 空则报错 400
    2. 创建 Order (status=PENDING)
    3. 遍历每件购物车商品：
       POST /products/{id}/reserve {quantity} → catalog 扣库存
       order.addLine(productId, name, quantity, price)
    4. 保存 Order 到数据库
    5. 清空购物车：DELETE FROM cart_items WHERE user_id=?
  ← 返回 Order 对象
  ← 前端购物车清空，提示 "Order created"
```

@Transactional 保证：扣库存、创建订单、清空购物车——要么全成功，要么全回滚。不会出现"库存扣了但订单没创建"的情况。

## 第 7 步：查看订单

[OrdersPage.tsx](frontend/src/pages/OrdersPage.tsx)

```
GET /orders?userId={id} → order-service → Order[]
  ← 前端渲染订单列表（状态/金额/时间/明细）
```

订单状态流转：PENDING → PAID → PROCESSING → SHIPPED → COMPLETED

## 关键架构要点

| 层级 | 技术 | 职责 |
|---|---|---|
| 前端 | React 18 + TypeScript + Vite | UI 渲染、状态管理、路由 |
| 数据管理 | TanStack Query 5 | 服务端数据缓存、自动刷新 |
| UI 组件 | Ant Design 5 | 表格、表单、卡片、提示 |
| HTTP 客户端 | Axios + 拦截器 | 统一 baseURL、自动带 JWT |
| 网关 | Spring Cloud Gateway MVC | 路由转发、CORS |
| 认证 | JWT + BCrypt | 无状态认证、密码加密 |
| 微服务 | Spring Boot 3 | 业务逻辑、数据持久化 |
| 数据库 | MySQL 8.4 | 用户、商品、订单、购物车 |
| 缓存 | Redis 7.4 | 会话、热点数据 |
| 容器化 | Docker Compose | 全栈一键启动 |
