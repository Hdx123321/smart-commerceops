================================================================
Product Image Storage (Docker Volume)
================================================================

Architecture:
  - Images uploaded via POST /api/admin/upload are saved to the catalog-service
    container at /app/uploads/images/products/<uuid>.<ext>.
  - The /app/uploads directory is backed by a named Docker volume:
    catalog-uploads (local) / smart-commerceops-main_catalog-uploads (cloud).
  - Database stores only the access path: /images/products/<uuid>.<ext>.
  - Frontend loads images via /images/... which nginx proxies to gateway,
    then gateway's catalog-images route proxies to catalog-service.
  - StaticResourceConfig serves /images/** from file:<uploadDir>/images/.

Design limitations (single-instance demo):
  - Images live on the Docker host's local disk. They are NOT replicated
    across multiple servers or regions.
  - If the Droplet is destroyed without a volume backup, images are lost.
  - For multi-instance production, migrate to object storage (S3/Spaces/R2).
  - This is NOT a long-term production architecture.

File type restrictions:
  - Allowed MIME types: image/jpeg, image/png, image/webp
  - Allowed extensions: .jpg, .jpeg, .png, .webp
  - Max file size: 5 MB per file
  - Max files per upload: 5

Cloud (Droplet) backup command:
  docker run --rm \
    -v smart-commerceops-main_catalog-uploads:/data \
    -v /root:/backup \
    alpine tar czf /backup/catalog-uploads-$(date +%Y%m%d).tar.gz -C /data .

Cloud (Droplet) restore command:
  docker compose -f docker-compose.ec2.yml stop catalog-service

  docker run --rm \
    -v smart-commerceops-main_catalog-uploads:/data \
    -v /root:/backup \
    alpine sh -c "cd /data && tar xzf /backup/catalog-uploads-YYYYMMDD.tar.gz"

  docker compose -f docker-compose.ec2.yml start catalog-service

Local backup command:
  docker run --rm \
    -v smart-commerceops_catalog-uploads:/data \
    -v /tmp:/backup \
    alpine tar czf /backup/catalog-uploads-$(date +%Y%m%d).tar.gz -C /data .

Local restore command:
  docker compose stop catalog-service

  docker run --rm \
    -v smart-commerceops_catalog-uploads:/data \
    -v /tmp:/backup \
    alpine sh -c "cd /data && tar xzf /backup/catalog-uploads-YYYYMMDD.tar.gz"

  docker compose start catalog-service

Migration reminder:
  When migrating the cloud database, you MUST also migrate the
  catalog-uploads volume. Otherwise product images will be missing.
  - Export DB: mysqldump (see above)
  - Export images: tar czf (see backup command above)
  - Copy both to new server
  - Import DB first, then restore images
